# Rules for Writing the Post

The rules (set in stone):

1. **No em dashes.** Ever.
2. **Write concisely.** Don't pad.
3. **Don't invent.** Only state what the data and project information support.
4. **Don't extrapolate.** Stick to what's documented.
5. **Feynman style.** First principles, clear explanations, no jargon for its own sake.
6. **Only write things you are 100% sure of.** If the data does not explicitly confirm it, do not say it. Do not assume. Do not guess. Do not add color.

## Style principles (discovered during editing)

7. **First-person agency.** The writer makes decisions; the tools execute them. "I fed each community to DeepSeek" lands. "DeepSeek generated titles" does not. The LLM is never the protagonist.

8. **The graph is the main character.** Every section orbits the same object. Extraction builds the graph. Audit corrects it. Build deploys it. The AI queries it. The reader never wonders what the subject is because everything returns to it.

9. **Structure reflects understanding.** Sentences flow in logical order. Each sentence builds on the last. The reader does not need to reassemble the pieces mentally because the order matches the logic.

10. **Respect the reader's intelligence.** Don't explain what the reader can infer. Don't repeat the same claim in two places. Padding insults the reader; conciseness trusts them.

11. **Remove the scaffolding.** Placeholders, notes-to-self, "improve this" — they are for the writer, not the reader. They break the illusion that the reader is getting a finished thought.

12. **Links are offers, not detours.** Links go at the exact moment the reader might want to go there. A link doesn't interrupt. It's an open door for the curious.

13. **Trust the data, hedge the claim.** Use "To my knowledge" before a big claim. Pin numbers to footnotes. The evidence carries the weight; the prose just points to it.

14. **One idea per beat.** The best sentences do one thing. "The model reasons; the graph provides the evidence." Not "The model reasons about the question while the graph simultaneously provides the evidence that supports the answer."
